# コンサルティング思考フレームワーク

> 作成日: 2025-12-02
> 更新日: 2025-12-02
>
> 本ドキュメントは、コンサルティングのプロが使う問題解決の思考法を体系的に整理したものです。
> ビジネスだけでなく、日常生活や学習における課題解決にも応用できる思考の「型」を、高校生にもわかるように具体例を交えて解説します。

---

## 1. 概要

コンサルティングにおける問題解決は、**思考法（プロセス）** と **思考ツール（道具）** の2つのレイヤーで構成されます。

料理に例えるなら、**思考法は「レシピ（料理の手順）」**、**思考ツールは「調理器具（包丁やフライパン）」** のような関係です。優れた料理人がレシピに沿って最適な調理器具を使い分けるように、優れた問題解決者は思考のプロセスに沿って適切なツールを使いこなします。

```
【思考法】＝ プロセス・流れ（レシピ）
    論点思考 → 仮説思考 → 戦略思考 → 戦術思考

【思考ツール】＝ 各プロセスで使う道具（調理器具）
    MECE、フェルミ推定、因数分解、ロジックツリー、etc.
```

## 2. 思考法の全体フロー

### 2.1 フロー図

問題解決は、基本的に以下の流れで進みます。

```
┌─────────────────────────────────────────────┐
│         論点思考（問いの設定）              │
│    「そもそも何を解くべきか？」            │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│         仮説思考（答えの仮置き）            │
│    「答えはこうではないか？」              │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│   ロジカルシンキング＋因数分解（検証設計）  │
│    「仮説を構造化し、検証可能な要素に分解」│
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│      フェルミ推定（定量的裏付け）           │
│    「規模感・数値を論理的に概算」          │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│         戦略思考（選択と集中）              │
│    「どこに資源を集中するか？」            │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│         戦術思考（実行設計）                │
│    「戦略をどう実現するか？」              │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│       実行・モニタリング・改善              │
│    「計画通りに進んでいるか？改善点は？」   │
└─────────────────────────────────────────────┘
```

### 2.2 重要な原則：論点が起点

**問いなき戦略は存在しない。** 論点（解くべき問い）がすべての出発点であり、戦略はその問いへの答えとして導かれます。

| 順序 | 理由 |
|------|------|
| 問いなき戦略は存在しない | 「何を達成すべきか」「何が問題か」という問いがなければ、戦略（打ち手）を立てる対象がない |
| 論点が戦略の質を決める | 間違った問い（例：売上不振の原因が商品力なのに「広告費を増やすべきか」と問う）に対して優れた戦略を立てても無意味 |
| 戦略は論点への回答 | 戦略とは「設定された論点に対する、選択と集中を伴う回答」である |

### 2.3 身近なテーマで思考法を体験する

この思考プロセスは、ビジネスだけでなく、学校生活にも応用できます。例えば、**『文化祭でクラスの出し物を成功させるには？』** というテーマで考えてみましょう。

| ステップ | 思考法 | 具体的なアクション |
|---|---|---|
| ① **問いの設定** | **論点思考** | 「そもそも『成功』とは何か？（目的：クラスの思い出作り？ 利益を最大化して打ち上げを豪華にすること？）」「本当にタピオカ屋がベストな選択か？」を考える。 |
| ② **答えの仮置き** | **仮説思考** | 目的を「利益の最大化」と設定し、「最近の流行を考えると、フルーツティーが一番売れるのではないか？」と仮の答え（仮説）を立てる。 |
| ③ **分解** | **因数分解** | 仮説を検証するために、売上を「売上 = 客数 × 客単価」という式に分解する。 |
| ④ **数値化** | **フェルミ推定** | 客数を概算してみる。「全校生徒800人 × 文化祭への来場率70% × ドリンク購入率30% = 約168人」のように、ざっくり計算して規模感をつかむ。 |
| ⑤ **方針決定** | **戦略思考** | 「競合の多いタピオカを避け、SNS映えする『光るフルーツティー』に予算を集中投資して差別化しよう」という大方針を決める。 |
| ⑥ **計画** | **戦術思考** | 「誰が・いつまでに・何をするか」を役割分担表（RACI）やスケジュール表（ガントチャート）で具体的に計画する。 |
| ⑦ **実行・改善** | **PDCA** | 計画通りに準備を進め、途中で問題が起きたら改善する。当日の売上を見て、価格や呼び込み方法を調整する。 |

## 3. 各思考法の詳細

### 3.1 論点思考

| 項目 | 内容 |
|------|------|
| **定義** | 問題解決において「本当に解くべき問い＝論点」を見極める思考法 |
| **問い** | 「何を解くべきか？」 |
| **役割** | 問題設定・課題の特定 |
| **アウトプット** | 解くべき問い |

#### 4つの基本プロセス

1. **課題の明確化** - いきなりHow（どうやるか）に飛びつかず、What（何を解くべきか）から着手
2. **論点候補の洗い出し** - MECE（モレなくダブりなく）で可能性を洗い出す
3. **優先順位の付与** - インパクト・実現可能性・スピードなどで絞り込み
4. **論点の確定・全体構造の見直し** - 因果関係や全体バランスを俯瞰

### 3.2 仮説思考

| 項目 | 内容 |
|------|------|
| **定義** | 限られた情報でも最初の答え（仮説）を立て、検証を繰り返して本質に迫る思考法 |
| **問い** | 「答えは何か？」 |
| **役割** | 解決策への効率的到達 |
| **アウトプット** | 仮説（仮の答え） |

#### 特徴

- 網羅的に情報収集してから考えるのではなく、結論（仮）を先に設定
- 仮説を証明・修正するために必要な情報だけを集める
- 仮説が間違っていればすぐ修正（試行錯誤のプロセス）

### 3.3 論点思考と仮説思考の関係

| 観点 | 論点思考 | 仮説思考 |
|------|----------|----------|
| **問い** | 「何を解くべきか？」 | 「答えは何か？」 |
| **役割** | 課題設定（問いの選択） | 解決策への近道（答えの仮置き） |
| **焦点** | 本当に考えるべき核心を絞る | 限られた情報で結論を先に立てる |

**論点思考が先、仮説思考が後** - 論点（問い）を間違えると、仮説検証の努力がすべて無駄になる。

### 3.4 戦略思考

| 項目 | 内容 |
|------|------|
| **定義** | 目的を達成するために、限られた資源（人、モノ、金、時間）をどこに集中させるかを決める思考法 |
| **問い** | 「どこで勝つか？」「何に集中するか？」 |
| **役割** | 方向性・資源配分の決定 |
| **アウトプット** | 戦略（基本方針） |

#### 特徴

- 将来を見据え、限られたリソースのどこに注力するかを決定
- 長期的・複合的な視点
- **やらないことを決める（選択と集中）**のが本質

### 3.5 戦術思考

| 項目 | 内容 |
|------|------|
| **定義** | 戦略を実行可能なアクションに落とし込む思考 |
| **問い** | 「どうやって勝つか？」「どう実現するか？」 |
| **役割** | 実行設計 |
| **アウトプット** | 実行計画 |

#### 戦略と戦術の違い

| 観点 | 戦略 | 戦術 |
|------|------|------|
| **問い** | どこで勝つか？ | どうやって勝つか？ |
| **視点** | 全体最適・中長期 | 部分最適・短期 |
| **抽象度** | 方向性・方針 | 具体的アクション |
| **例** | 「ベトナム市場に参入する」 | 「ホーチミンに営業拠点を設置し、Web広告を展開する」 |

## 4. 思考ツールの詳細

### 4.1 思考ツールの位置づけ

```
┌─────────────────────────────────────────────────────────────┐
│                        MECE（全工程の基盤）                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  論点思考 ──→ 仮説思考 ──→ 戦略思考 ──→ 戦術思考        │
│     │            │            │            │              │
│     ↓            ↓            ↓            ↓              │
│  イシュー     因数分解      3C/SWOT      WBS             │
│  ツリー      ロジック       5Forces     ガント           │
│             ツリー                       RACI            │
│               ↓                                          │
│          フェルミ推定                                     │
│          （定量化）                                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 MECE（ミーシー）

| 項目 | 内容 |
|------|------|
| **役割** | **モレなく、ダブりなく**物事を整理するための基本原則 |
| **使う場面** | 論点洗い出し、因数分解、ロジックツリー作成時など、あらゆる場面で使う |
| **位置** | **全フェーズで横断的に使用**（最も基礎的なツール） |
| **具体例** | 「今日の夕飯の買い物リスト」を考えるときに、「肉類、野菜、その他」と分類するイメージ。これにより買い忘れ（モレ）や、同じものを二重でリストアップ（ダブり）するのを防ぐ。 |

### 4.3 因数分解

| 項目 | 内容 |
|------|------|
| **役割** | 複雑な問題を構成要素に分解して、考えやすくする |
| **使う場面** | 仮説の検証設計、KPI設計、原因分析 |
| **位置** | **仮説思考の直後**（検証可能にするため） |

#### 例：「ASEAN市場が有望」の因数分解

漠然とした仮説を、検証可能な具体的な要素に分解します。

```
「ASEAN市場が有望」
    ├── 市場規模（十分な規模があるか？）
    ├── 成長率（今後伸びるか？）
    ├── 競合状況（勝てる余地があるか？）
    ├── 参入障壁（自社が参入可能か？）
    └── 収益性（利益が出るか？）
```

### 4.4 フェルミ推定

| 項目 | 内容 |
|------|------|
| **役割** | 未知の数値を、いくつかの根拠を元に論理的に概算（ざっくり計算）する |
| **使う場面** | 仮説の妥当性検証、市場規模推定、投資判断 |
| **位置** | **仮説思考〜戦略思考の間**（定量的な裏付けのため） |

#### 例：ベトナム市場規模の推定

```
仮説思考：「ベトナムのEC市場が有望ではないか？」
      ↓
因数分解：市場規模 = 対象人口 × EC利用率 × １人あたり年間購入額
      ↓
フェルミ推定：「ベトナムのEC市場規模は？」
      │
      ├── 人口1億人 × EC利用可能層30% × EC利用率50% × 年間購入額3万円
      │   = 4,500億円
      ↓
戦略思考：「4,500億円の市場でシェア1%でも45億円。参入価値あり」
```

### 4.5 ロジックツリー

| 項目 | 内容 |
|------|------|
| **役割** | 問題の原因や解決策を、MECEを意識しながらツリー状に分解・整理する |
| **使う場面** | 論点の構造化、原因究明、打ち手の網羅 |
| **位置** | **論点思考〜仮説思考** |

### 4.6 戦術設計のツール

| ツール | 用途 | 内容 |
|--------|------|------|
| **WBS** | タスク分解 | **W**ork **B**reakdown **S**tructure の略。プロジェクトのタスクを細かい作業に分解したリスト。 |
| **ガントチャート** | スケジュール設計 | 誰が・いつまでに・何をやるかを視覚的に示した工程表。 |
| **RACI表** | 責任分担 | **R**esponsible(実行者), **A**ccountable(責任者), **C**onsulted(協業者), **I**nformed(報告先)を明確にする表。 |
| **KPI設計** | 成果指標 | **K**ey **P**erformance **I**ndicator の略。目標の達成度を測る具体的な数値目標。（例：テストで80点取る） |

## 5. 全体像まとめ

### 5.1 フェーズ別一覧

| フェーズ | 思考法 | 使う思考ツール | 問い | アウトプット |
|----------|--------|----------------|------|--------------|
| 問題設定 | 論点思考 | MECE、イシューツリー | 何を解くか？ | 解くべき問い |
| 方向仮置き | 仮説思考 | アナロジー（類推）、アブダクション（仮説形成） | 答えは何か？ | 仮説 |
| 構造化・分解 | ロジカルシンキング | MECE、因数分解、ロジックツリー | どう分解するか？ | 検証項目 |
| 定量化・推定 | フェルミ推定 | 因数分解、概算 | 規模感は？ | 数値の目安 |
| 方針決定 | 戦略思考 | 3C分析、SWOT分析など | どこに集中？ | 戦略 |
| 実行設計 | 戦術思考 | WBS、ガントチャート、RACI、KPI | どう実現？ | 実行計画 |
| 実行・改善 | PDCA/OODA | KPIモニタリング | うまくいってる？ | 成果・学習 |

### 5.2 思考法と思考ツールの区分

| 区分 | 名称 | 性質（料理の例） |
|------|------|------|
| **思考法（プロセス）** | 論点思考、仮説思考、戦略思考、戦術思考 | 「流れ」を規定する（**レシピ**） |
| **思考ツール（道具）** | MECE、因数分解、フェルミ推定、ロジックツリー | 各プロセスで使う「武器」（**調理器具**） |

## 6. 4層構造による分類（より深い理解のために）

ここまでの思考フレームワーク全体を、**「家づくり」**に例えて4つの層で整理してみましょう。この分類を知ることで、何を優先的に学ぶべきかが明確になります。

- **メンタル・素養（土地・基礎工事）**：家を建てるための頑丈な土地。
- **思考技術（設計図）**：どんな家を建てるかの設計プロセス。
- **リサーチ・知識アップデート（建材・工法）**：家を建てるための材料や技術。
- **武器・具体知識（大工道具）**：設計図を形にするための具体的な道具。

> **Note**: メタ認知/メタ思考は本フレームワークの「外側」で常時稼働し、思考プロセス全体を監視・調整します。
> 詳細は[メタ思考ドキュメント](./meta-thinking.md)を参照してください。

### 6.1 本フレームワークの使い方：メタ認知との関係

本フレームワーク（4層構造）は、**1回の思考プロセス**で使う型・ツールを提供します。
実際の問題解決では、**メタ認知/メタ思考**と組み合わせて使います。

```
┌─────────────────────────────────────────────────────────────┐
│              アジャイル（反復プロセス全体）                  │
│                Sprint 1 → Sprint 2 → Sprint 3 → ...        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   各粒度（Epic / Story / Task）ごとに：                    │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐   │
│   │  メタ認知/メタ思考（常時稼働）                      │   │
│   │  「今の思考・判断は正しいか？」を問い続ける        │   │
│   │  → 詳細は meta-thinking.md                         │   │
│   ├─────────────────────────────────────────────────────┤   │
│   │  本フレームワーク（1回の思考プロセス）             │   │
│   │  論点→仮説→戦略→戦術                              │   │
│   │  ＋ 4層構造（素養/技術/リサーチ/武器）            │   │
│   └─────────────────────────────────────────────────────┘   │
│                                                             │
│   → 粒度を下げる（解像度を上げる）ごとに繰り返す         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 4層構造の全体図

```
┌─────────────────────────────────────────────────────────────┐
│               メンタル・素養（土台）                        │
│         MECE意識、仮説志向、検証マインド...                │
│                   【時代遅れになりにくい】                  │
├─────────────────────────────────────────────────────────────┤
│               思考技術（エンジン/設計図）                   │
│       論点思考→仮説思考→戦略思考→戦術思考                │
│                   【時代遅れになりにくい】                  │
├─────────────────────────────────────────────────────────────┤
│         リサーチ・知識アップデート（横断層）                │
│     情報収集、AI活用、最新知識の取得・評価                 │
│     → 全フェーズで常時稼働、思考の質を支える              │
│                   【能力は残るが、やり方は新しくなる】      │
├─────────────────────────────────────────────────────────────┤
│               武器・具体知識（出力装置/道具）               │
│     フレームワーク、ツール、業界知識、技術知識             │
│                   【時代遅れになりやすい→常に更新が必要】  │
└─────────────────────────────────────────────────────────────┘
```

### 6.3 4層の定義と特性

| 層 | 名称 | 定義 | 時代遅れになるか |
|----|------|------|------------------|
| **土台** | メンタル・素養 | 思考を支える「姿勢」「習慣」「マインドセット」 | なりにくい |
| **エンジン** | 思考技術 | 思考の「型」「プロセス」 | なりにくい |
| **横断層** | リサーチ・知識アップデート | 情報収集・学習・評価の活動 | **能力は残るが、やり方は新しくなる** |
| **出力装置** | 武器（ツール・具体知識） | 具体的な「道具」「フレームワーク」「専門知識」 | **なりやすい→常に更新が必要** |

### 6.4 メンタル・素養（土台）

思考の基盤となる姿勢・習慣。**時代遅れになりにくく**、一度身につければ一生使える資産です。

| 名称 | 内容 |
|------|------|
| **MECE意識** | 常にモレ・ダブりを意識する習慣 |
| **仮説志向** | 情報が不完全でも仮の答えを持つ姿勢 |
| **構造化志向** | 物事を分解・整理して捉える習慣 |
| **検証マインド** | 仮説を事実で検証し修正する姿勢 |
| **選択と集中の覚悟** | 「やらないこと」を決断できるマインド |
| **問い続ける姿勢** | 「本当の問題は何か」を問い続ける習慣 |

### 6.5 思考技術（エンジン）

思考の型・プロセス。これも**時代遅れになりにくく**、普遍的に適用可能です。

| 名称 | 役割 |
|------|------|
| **論点思考** | 何を解くべきかを決める |
| **仮説思考** | 答えを先に仮置きして検証する |
| **ロジカルシンキング** | 論理的に構造化・整合性をとる |
| **戦略思考** | どこに集中するかを決める |
| **戦術思考** | どう実行するかを設計する |

### 6.6 リサーチ・知識アップデート（横断層）

全ての思考フェーズに横断的に関わる活動。リサーチ能力自体は残りますが、手法・ツール（例：AIの活用法）は常に新しくなります。

#### 各フェーズでのリサーチの役割

| フェーズ | リサーチの目的 | 例 |
|----------|----------------|-----|
| **論点思考** | 問いの妥当性確認 | 「この問いは今も有効か？市場環境は変わっていないか？」 |
| **仮説思考** | 仮説の精度向上 | 「類似事例は？最新の競合動向は？」 |
| **構造化・定量化** | 検証データ収集 | 「最新の市場データ、技術トレンドは？」 |
| **戦略思考** | 選択肢の網羅性確認 | 「新たな参入障壁、規制変更はないか？」 |
| **戦術思考** | 実行手段の最新化 | 「より効率的なツール、手法はないか？」 |

### 6.7 武器・具体知識（出力装置）

具体的なフレームワーク・ツール・専門知識。**時代遅れになりやすいため、常に新しいものを学び続ける必要があります。**

#### 時代遅れになる速さによる分類

| 分類 | 例 | 速さ | 対応 |
|---|---|---|---|
| **普遍的フレームワーク** | MECE, 因数分解, ロジックツリー | 遅い | 基礎として習得 |
| **戦略フレームワーク** | 3C, SWOT, 5Forces | 普通 | 限界を理解して使用 |
| **ツール・技術** | AI, SaaS, 分析ツール | 速い | 常にアップデート |
| **具体知識** | 業界動向, 規制, 技術トレンド | とても速い | リサーチで常時更新 |

#### フェーズ別の武器一覧

| フェーズ | 武器の例（一部） |
|---|---|
| **問題設定** | イシューツリー |
| **構造化・分解** | 因数分解、ロジックツリー |
| **定量化** | フェルミ推定 |
| **戦略立案** | 3C（顧客・自社・競合）、SWOT（強み・弱み・機会・脅威）、5Forces、PPM |
| **実行設計** | WBS、ガントチャート、RACI |
| **成果管理** | KPI、OKR |
| **改善サイクル** | PDCA、OODA |

### 6.7 陳腐化の全体像（まとめ）

```
【時代遅れになりにくい】
    ↑
    │  メンタル・素養（MECE意識、仮説志向...）
    │  思考技術（論点思考、仮説思考...）
    │
────┼──────────────────────────────────────────
    │  リサーチ能力（情報収集・評価スキル）← 能力は残るが、やり方は変わる
    │
────┼──────────────────────────────────────────
    │  武器・具体知識
    │    ├── 普遍的フレームワーク（MECE, 因数分解...）← 比較的安定
    │    ├── 戦略フレームワーク（3C, SWOT...）← 普通
    │    ├── ツール・技術（AI, SaaS...）← 時代遅れになりやすい
    │    └── 具体知識（業界知識, 技術知識...）← 非常に時代遅れになりやすい
    ↓
【時代遅れになりやすい】

※ メタ認知/メタ思考は本フレームワークの「外側」で常時稼働
```

### 6.8 実務へのヒント：学び方の方針

| 層 | 学び方の方針 |
|----|----------|
| **メンタル・素養** | 若いうちに徹底的に身につける。一生使える資産になる。 |
| **思考技術** | 基礎として習得し、実践で磨き続ける。 |
| **リサーチ能力** | 情報の正しさを見抜く力を鍛える。ツールは都度、新しいものを習得する。 |
| **武器・具体知識** | 古くなることを前提に、常にアップデートする。AIや最新ツールを積極的に活用する。 |

> **Note**: メタ認知/メタ思考の学び方については[メタ思考ドキュメント](./meta-thinking.md)を参照。

## 7. 具体例：新規市場参入（ビジネス版）

| ステップ | 思考法/ツール | 内容 |
|----------|---------------|------|
| ① | **論点思考** | 「会社の成長のために新規市場に参入すべきか？参入するならどこに？」 |
| ② | **仮説思考** | 「経済成長が著しいベトナム市場が最適ではないか？」 |
| ③ | **因数分解** | 市場規模・成長率・競合・参入障壁・収益性に分解して検討項目を整理 |
| ④ | **フェルミ推定** | 市場規模を概算（人口×対象層×購買率×単価） |
| ⑤ | **ロジカルシンキング** | 各要素をMECEで整理し、因果関係（儲かるか、勝てるか）を確認 |
| ⑥ | **検証** | 分解した各要素を、実際のデータ（統計、レポートなど）で検証 |
| ⑦ | **戦略思考** | 「ベトナムに3年で10億投資し、高所得者層向けにシェア5%獲得を目指す」と方針を決定 |
| ⑧ | **戦術思考** | WBS/ガントチャート/RACIで具体的な実行計画を策定 |
| ⑨ | **実行・改善** | PDCA/OODAサイクルで進捗を管理し、状況に応じて計画を修正 |
