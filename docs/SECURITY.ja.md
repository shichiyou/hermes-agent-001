# セキュリティポリシー

## 脆弱性の報告方法

疑いのあるセキュリティ脆弱性についてパブリックなイシューを開かないでください。

代わりに、このリポジトリで有効になっている場合は GitHub のプライベート脆弱性報告フローを使用してください。
そのフローが利用できない場合は、GitHub のリポジトリ所有者に非公開で連絡し、以下を含めてください：

- 問題の明確な説明
- 影響を受けるファイルまたはコマンド
- 再現手順
- わかっている場合は影響評価

## 対応に関する期待

メンテナーは時間の許す限り報告をレビューし、澄清や最小限の再現物を求める場合があります。

## 範囲に関する注記

このリポジトリは開発環境ベースラインです。セキュリティに関連する報告には以下が含まれる場合があります：

- 安全でないデフォルトのコンテナ動作
- スクリプトまたはドキュメントでの認証情報取り扱いミス
- セットアップスクリプトでのコマンドインジェクションまたは権限昇格パス
- インストールまたはアップデートフローによって導入されたサプライチェーンリスク

このベースラインが直接的にリスクを導入しない限り、このリポジトリ外のベンダー固有サービスの動作は Generally out of scope です。
