# Workspace Hygiene — Parent Repository Pollution Prevention

## Problem

Hermes Agent generates files during Discord/Cron tasks (research reports, audit reports, cloned repos).
When these are written inside the parent repository (`/workspaces/hermes-agent-lab/`), they pollute
the repo and get accidentally committed. This skill enforces output routing and pre-commit checks.

## Output Routing Rules

| Artifact Type | Destination | Example |
|---|---|---|
| Research reports | `~/workspace/reports/` | `~/workspace/reports/aidlc-workflows-research-report.md` |
| Security/audit reports | `~/workspace/audits/` | `~/workspace/audits/discord-security-audit-report.md` |
| Cloned external repos | `~/workspace/repos/` | `~/workspace/repos/aidlc-workflows/` |
| Temporary / scratch files | `~/workspace/tmp/` | `~/workspace/tmp/compare-output.txt` |
| Wiki content (persistent) | `~/wiki/` (submodule) | `~/wiki/entities/foo.md` |
| App code / project config | Parent repo root | Intentional changes only |

## Pre-Commit Checklist (MUST run before every `git add` in the parent repo)

1. **Path check**: Is the file inside `/workspaces/hermes-agent-lab/`?
2. **Intent check**: Is this an intentional change to the repo (code, config, docs), OR an agent-generated artifact?
3. **Redirect**: If artifact -> write to `~/workspace/` instead, do NOT stage it.
4. **Naming guard**: Files matching these patterns in the repo root MUST NOT be committed:
   - `*-research-report.md`
   - `*-audit-report.md`
   - `*-security-audit-report.md`
   These are covered in `.gitignore` as a safety net, but do not rely on that alone.

## What Belongs in the Parent Repo

**OK to commit** — intentional project changes:
- Source code and application files
- Project documentation (README, CHANGELOG, CONTRIBUTING, etc.)
- Configuration and tooling files
- Wiki submodule pointer

**NEVER commit** — agent-generated artifacts:
- Research reports generated by agent tasks
- Audit/security reports generated by agent tasks
- Cloned external repositories (even for research)
- Scratch/comparison/temporary files

## Cleanup Procedure

If agent-generated files are found in the parent repo:

```bash
# Move to correct location
mv /workspaces/hermes-agent-lab/<file>-research-report.md ~/workspace/reports/
mv /workspaces/hermes-agent-lab/<file>-audit-report.md ~/workspace/audits/

# Remove from git tracking
cd /workspaces/hermes-agent-lab
git rm <file>

# Commit the cleanup
git commit -m "cleanup: move agent artifact to ~/workspace/"
```

## Devcontainer Persistence Note

`~/workspace/` lives in the home directory (`/home/vscode/workspace/`). In the current devcontainer
setup, `/home/vscode` is a named volume — it survives container restarts but is destroyed by
`docker compose down -v`. If persistence across full teardown is needed, use the wiki submodule
or push to an external repository.

## Verification

After any file write or git operation in the parent repo, verify:

```bash
# Check for agent artifacts in repo root
ls /workspaces/hermes-agent-lab/*-report.md /workspaces/hermes-agent-lab/*-audit.md 2>/dev/null
# Should return nothing. If files found -> move to ~/workspace/
```
