## Wiki操作の指示パターン（ユーザー指定方法）

| 指示 | 動作 |
|---|---|
| 「〇〇について調べて」「〇〇を教えて」 | 回答のみ。Wikiには書き込まない |
| 「〇〇について調べてWikiに追加」「〇〇をWikiに取り込んで」 | 調査 → Wikiページ作成 → index/log更新 → git commit |
| 「〇〇について調べて、Wikiに追加するか提案して」 | 調査 → 追加するか確認 → 承認後に追加 |
| 「このURLを取り込んで」 | Ingest操作（raw保存→ページ作成→相互リンク→index/log更新） |

明示的に「Wikiに追加」「Wikiに取り込み」等の指示がない場合は、回答のみでWikiには書き込まない。
§
ユーザーはAIが「成功したはずのストーリー」を先に構築し、物理的証拠（git status, logs等）を軽視または無視して報告する「妄想壁（Hallucination/Confirmation Bias）」を極めて強く警戒している。
AIが「改善できる」という根拠のない自信を述べることよりも、不都合な事実（失敗）を正確に認め、物理的エビデンス（Raw Output）のみに基づいて現状を報告することを最重視する。
「物理的証拠を提示せずに完了を主張すること」は、能力不足ではなく「誠実性の欠如」とみなされる。
